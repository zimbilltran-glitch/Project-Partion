# FINSANG v2.0 — Vietnamese Financial Statement Pipeline & Terminal

**Finsang** is a production-grade financial data pipeline for Vietnamese public companies.
It extracts corporate financial statements (BCTC) from the Vietcap API, normalizes data into
a strict accounting schema, stores it as encrypted Parquet files, and serves analytics via
both a terminal UI and a professional web dashboard.

## Architecture: B.L.A.S.T. Framework

```
[B]lueprint → golden_schema.json (345 fields mapped to TT200 standard)
[L]ink      → Vietcap API extraction via Provider pattern
[A]rchitect → Hive-partitioned, encrypted Parquet storage
[S]tylize   → Rich terminal UI + React web dashboard
[T]rigger   → Automated pipeline with Supabase cloud logging
```

## Quick Start

### 1. Clone & Setup

```bash
git clone <repo-url>
cd Finsang
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
```

### 2. Configure Environment

Create a `.env` file in the project root:

```env
# Supabase (required for cloud sync + web dashboard)
SUPABASE_URL="https://your-project.supabase.co"
SUPABASE_KEY="your-supabase-service-role-key"

# Pipeline config
FINSANG_TICKERS=VHC,FPT,HPG
FINSANG_VIEW_PERIODS=quarter

# Auto-generated on first run (do NOT commit)
# FINSANG_ENCRYPTION_KEY=<auto-generated-fernet-key>
```

For the frontend, create `frontend/.env`:

```env
VITE_SUPABASE_URL="https://your-project.supabase.co"
VITE_SUPABASE_ANON_KEY="your-supabase-anon-key"
```

### 3. Run the Pipeline

```bash
# Single ticker
python Version_2/pipeline.py --ticker VHC

# All configured tickers
python Version_2/run_all.py
```

### 4. View Data

#### Terminal Viewer (Rich)
```bash
python Version_2/viewer.py --ticker VHC --sheet cdkt --period year
python Version_2/viewer.py --ticker VHC --sheet kqkd --period quarter
python Version_2/viewer.py --ticker VHC --sheet cstc --period year
```

Available sheets: `cdkt` (Balance Sheet), `kqkd` (Income Statement),
`lctt` (Cash Flow), `note` (Footnotes), `cstc` (Financial Ratios)

#### Web Dashboard (React + Supabase)
```bash
# Step 1: Sync data to Supabase
python Version_2/sync_supabase.py --all

# Step 2: Start frontend dev server
cd frontend && npx vite --host
```

## Security

- **Encryption at Rest**: All Parquet files are encrypted with Fernet (AES-128-CBC).
  The key is auto-generated and stored in `.env` (`FINSANG_ENCRYPTION_KEY`).
- **Zero Hardcoded Secrets**: All credentials managed via `.env` (gitignored).
- **Supabase RLS**: Row Level Security enabled on all tables.
  Write access restricted to `service_role`; public `anon` key is read-only.
- **CI Security Scanning**: `bandit` runs on every push to catch vulnerabilities.

## Data Flow

```

                                    ▼

```
